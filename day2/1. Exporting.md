#### Exporting with `"type": "module"`
- There are two ways to export functions from a file: `export` and `export default` 
- NOTE: must use `"type": "module"` in `package.json` for this to work 

#### `export` (AKA Named Exports) 
- Allows us to export multiple things
- Must import functions using the exact same name 
```js
// someFileOne.js
export function functionOne() {
  console.log("Hello from function one!");
}

export function functionTwo() {
  console.log("Hello from function two!");
}
```
```js
// index.js
import { functionOne, functionTwo } from './someFileOne.js';
functionOne();
functionTwo();
``` 

#### `export default` (Default Export)
- Only one default export per file!
- You can import that function with any name you want
- No `{}` needed when importing 
```js
// someFileTwo.js
export default function functionOne() {
  console.log("Hello from default function!");
}
```
```js
// index.js
import defaultFunction from './someFileTwo.js';
defaultFunction();
```
---
#### Exporting with `"type": "commonjs"`
- `export` / `export default` will NOT work
- Must use older syntax: 
```js
// someFileThree.js
function commonJSFunction() {
  console.log("Hello from commonJS!");
}

module.exports = commonJSFunction;
```
```js
// index.js
const func = require('./someFileThree.js');
func();
```
